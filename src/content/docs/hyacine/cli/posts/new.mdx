# 创建新文章

`new` 命令用于快速创建新的 Markdown 文章。系统会自动生成规范的文件名、slug，并将文章元数据保存到数据库。

## 基本用法

```bash
hyc new [title]
```

## 参数说明

| 参数 | 说明 | 必需 |
|------|------|------|
| `[title]` | 文章标题 | 否 |

如果未提供标题，系统会以交互式提示要求输入。

## 功能特性

✅ **智能文件名生成** - 根据标题自动生成规范的文件名
✅ **唯一性检查** - 防止创建重复标题的文章
✅ **自动 Slug 生成** - 创建 URL 友好的唯一标识符
✅ **内容哈希计算** - 用于后续的变更追踪
✅ **自动创建资源目录** - 如启用了资源文件夹模式(.hexo-assetdir)

## 使用示例

### 直接创建

```bash
# 使用命令行参数创建
hyc new "我的第一篇文章"

# 使用多个单词（自动连接）
hyc new 使用 Hyacine CLI 的最佳实践
```

### 交互式创建

```bash
# 运行命令后会提示输入标题
hyc new
# 输入: 如何优化数据库性能
```

## 创建的文件内容

新文章会包含基础博客元数据：

```markdown
title: ${title}
date: ${dateStr}
categories: []
---

# ${title}

在这里开始写作...
```
