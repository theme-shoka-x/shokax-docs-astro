# 编辑文章

`edit` 命令在您的默认编辑器中打开文章，让您快速开始编写或修改内容。

## 基本用法

```bash
hyc edit <search>
```

## 参数说明

| 参数 | 说明 | 必需 |
|------|------|------|
| `<search>` | 查询关键词（标题、slug 或路径） | 是 |

## 工作原理

1. **解析查询** - 根据搜索词在数据库中查找文章
2. **定位文件** - 获取文章的完整文件路径
3. **打开编辑器** - 使用系统默认编辑器打开文件

## 支持的编辑器

系统会自动检测并使用您系统的默认编辑器：

- **Windows** - 关联的默认应用（通常是记事本、VS Code 等）
- **macOS** - 通过 `open` 命令调用默认代码编辑器
- **Linux** - 通过 `xdg-open` 命令调用默认编辑器

## 使用示例

### 按标题打开

```bash
hyc edit "我的第一篇文章"
hyc edit "第一篇"
```

### 按路径模糊匹配打开

```bash
hyc edit "posts/my-first"
```

## 技巧提示

:::tip
如果您使用的是 VS Code，编辑器会自动在当前窗口或新窗口中打开文件，取决于您的 VS Code 设置。
:::

:::tip
编辑文章时可以直接修改 YAML 前置元数据（title、draft、categories等），保存文件后通过 `hyc sync` 命令同步到数据库。slug 由系统自动管理，不包含在文章的 frontmatter 中。
:::

## 联动命令

```bash
# 编辑后同步数据库
hyc edit "article"
# ...修改文件...
hyc sync

# 或编辑后直接发布
hyc edit "article"
# ...修改完成...
hyc publish "article"
```
