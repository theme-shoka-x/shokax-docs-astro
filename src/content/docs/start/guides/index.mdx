## 环境要求

要使用 ShokaX Astro，你需要安装：

- [Bun](https://bun.sh/) 作为 JS 运行时
- [Git](https://git-scm.com/) 作为项目克隆和版本管理工具

如果你没有听说过这些程序或还没有安装，请转到侧边栏中的“术语表”部分来查看其介绍和安装指导

## 快速上手

### 使用 HyC 进行交互式安装（推荐）

```bash
bun install @hyacine/cli -g

hyc -V
# 如果有输出，证明安装成功

# 选一个好位置
mkdir astro-blog-shokax # 也可以是别的名字，或者使用文件管理器自行创建一个空文件夹
cd astro-blog-shokax # 切换到你准备安装 ShokaX 的文件夹

hyc serve --setup
# 安装程序会自动启动，输出类似于：
# ✅ 服务已在 3789 端口开放
# 🔐 认证码（高敏感，请勿泄露）: <16位的认证码>
# 🌐 请前往官方控制台进行操作：https://hyc.kaitaku.xyz/
# 浏览器打开提示的官方控制台，即------^^^^^^^^^^^^^^^^^
# 在控制台中输入16位认证码，随后进入部署流程
```

随后，按照网页的指导进行安装即可，网页会：
1. 自动确认依赖安装是否正确
2. 自动克隆 github 储存库，并根据网络情况自动使用镜像源和加速源
3. 自动安装 Bun 依赖，并根据网络情况自动使用镜像源

结束后，网页会提示你执行 HyC 的初始化工作并重启服务器到正常模式

:::tip
Setup 模式在 Clone 完成后无法二次进入，如果部署流程中途中断请删除当前文件夹并重启程序
:::

### 基于 Git 的安装

打开你系统中的命令行工具，然后运行：

```bash
# 记得先切换到你准备放置这个 Blog 的地方

git clone https://github.com/theme-shoka-x/astro-blog-shokax.git

cd ./astro-blog-shokax

bun install
bun run dev
```

开发服务器会自动启动，你可以访问提示的网址（通常为`http://localhost:4321`）来浏览你的网站

### 更新主题

<Aside type="tip">
  请务必经常更新 ShokaX Astro，ShokaX Astro 目前仍处于积极开发阶段中，会经常添加新功能和实施改进
</Aside>

在 ShokaX Astro 发布新版本时，你可以通过 Git 更新主题：

```bash
git pull

bun install
# 更新完成
```

如果想个性化和定制化 ShokaX，建议你 Fork 我们的仓库，这能极大降低后续更新的难度，方法可见左侧的“最佳实践”栏目（此部分正在编写中，暂不可用）
